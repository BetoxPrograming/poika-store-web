# Lesson 01 — Project Creation, TypeScript Setup and GitHub Repository

## Lesson goal

The goal of this lesson is to create the base project for **Póika Store Web**.

This first lesson does not focus on the visual design of the store yet. Its purpose is to prepare the development environment, create the frontend project with React and TypeScript, understand the basic files generated by Vite, run the application locally and publish the first version of the project in GitHub.

At the end of this lesson, the project should exist as a working web application that can be opened in the browser and tracked with Git.

* * *

## Main concept: project bootstrap

A project bootstrap is the initial setup of a software project.

In this stage, the developer creates the base structure that will support the rest of the application. This includes the project folder, package configuration, source files, dependencies, development commands and version control.

The flow of this lesson is:

    Empty folder
        → Vite React TypeScript project
        → Local development server
        → First application screen
        → Git repository
        → GitHub repository

This is important because every future lesson depends on this base.

If the project is not created correctly, later steps such as layouts, translations, API communication, database data and deployment become harder to organize.

## Node.js

Node.js is the runtime that allows JavaScript and TypeScript development tools to run on the computer.

React code runs in the browser, but the tools used to create, compile and serve the project run with Node.js.

For this project, Node is needed to:

- Create the Vite project.
- Install dependencies.
- Run the development server.
- Execute build commands.
- Run linting tools.

Node is not the same as the browser.

The browser executes the final frontend code. Node executes the development tools that prepare that frontend code.

Useful command:

    node -v

This command confirms that Node is installed.

* * *

## npm

npm is the package manager used in this project.

A package manager is a tool that downloads and manages libraries. In this project, npm is responsible for installing React, TypeScript, Vite and all the development dependencies.

npm is also used to execute project scripts.

Useful command:

    npm -v

This command confirms that npm is installed.

In this project, npm is important because the application is controlled through the scripts defined in `package.json`.

* * *

## Vite

Vite is the tool used to create and run the frontend project.

Vite provides a fast development server and prepares the project for production builds.

In a React TypeScript project, Vite gives the initial structure, the configuration files and the commands needed to run the application.

The most important idea is:

    Vite is not the application.
    Vite is the tool that helps develop and build the application.

The application itself is written with React and TypeScript.

* * *

## React

React is the library used to build the user interface.

Instead of writing one large HTML file, React divides the interface into components.

A component is a reusable piece of the screen.

Examples of components that will appear later in the project are:

    Header
    Footer
    Layout
    LanguageSwitcher
    HomePage

In this first lesson, the most important component is:

    App.tsx

`App.tsx` represents the main component of the application.

* * *

## TypeScript

TypeScript is a typed version of JavaScript.

JavaScript allows code to run without declaring many types explicitly. TypeScript adds a type system that helps detect errors before running the project.

For example, TypeScript can help detect when a function receives the wrong type of value.

The main benefit is that the code becomes more predictable and easier to maintain.

In this project, files that contain TypeScript and React usually use the extension:

    .tsx

The `.tsx` extension means that the file can contain both TypeScript logic and JSX markup.

* * *

## JSX and TSX

JSX is the syntax that allows React to write HTML-like structures inside JavaScript or TypeScript.

Example:

    function App() {
      return <h1>Póika Store</h1>
    }

This looks like HTML, but it is written inside a TypeScript file.

When TypeScript and JSX are used together, the file extension is `.tsx`.

That is why React components in this project use names like:

    App.tsx
    main.tsx

* * *

## package.json

`package.json` is one of the most important files in a Node and TypeScript project.

It describes the project and defines:

- The project name.
- The project version.
- The scripts that can be executed.
- The dependencies required by the application.
- The development dependencies used by the tools.

The main scripts are:

    npm run dev
    npm run build
    npm run lint
    npm run preview

Each script has a different purpose.

`npm run dev` starts the local development server.

`npm run build` creates the production version of the project.

`npm run lint` checks the code for style or quality problems.

`npm run preview` allows testing the production build locally.

* * *

## index.html

`index.html` is the HTML entry file of the application.

Even though this is a React project, the browser still needs an HTML file to start.

The most important part of this file is the root element:

    <div id="root"></div>

React uses this element as the place where the application will be mounted.

This means that React takes control of that part of the page and renders the components inside it.

* * *

## src/main.tsx

`main.tsx` is the TypeScript entry point of the React application.

Its job is to connect React with the HTML file.

The basic responsibility of this file is:

    Find the root element in index.html
        → Create the React root
        → Render the App component

Conceptually, the flow is:

    index.html
        → main.tsx
        → App.tsx

This file is important because it starts the React application.

* * *

## src/App.tsx

`App.tsx` is the first main component of the application.

In the beginning, Vite creates a demo version of this file. That demo usually includes logos, buttons and example content.

For Póika Store, that example content must be replaced with content that belongs to the store.

The purpose is to stop having a generic Vite page and start having a real Póika page.

At this stage, the page can be simple. The important part is that it belongs to the project.

* * *

## CSS files

The project starts with basic CSS files:

    App.css
    index.css

`App.css` is usually related to the styles of the App component.

`index.css` is usually used for global styles.

Global styles affect the general behavior of the page, such as margins, font rendering or page sizing.

In later lessons, these files are cleaned and adapted to work with Tailwind CSS.

## tsconfig files

The project includes TypeScript configuration files.

These files tell TypeScript how to analyze and compile the project.

The common files are:

    tsconfig.json
    tsconfig.app.json
    tsconfig.node.json

The main idea is that TypeScript needs rules to understand how strict the project should be and which files should be included.

In this lesson, these files are not the main focus, but they are part of the foundation of the project.

* * *

## vite.config.ts

`vite.config.ts` contains the Vite configuration.

This file tells Vite how the project should behave.

In a basic React project, Vite uses this file to activate the React plugin and prepare the development environment.

The important idea is:

    package.json controls the commands.
    vite.config.ts controls the Vite behavior.

* * *

## .gitignore

`.gitignore` tells Git which files or folders should not be saved in the repository.

This is important because not every local file belongs in GitHub.

Examples of files or folders that should not be committed are:

    node_modules/
    dist/
    .env

`node_modules` is ignored because it can be recreated with:

    npm install

`dist` is ignored because it is generated by the build process.

`.env` is ignored because it may contain private configuration.

* * *

## Git

Git is the version control system used to track the history of the project.

It allows the developer to save changes through commits.

A commit is a snapshot of the project at a specific moment.

The basic flow is:

    Modify files
        → Check status
        → Add files
        → Create commit
        → Push to GitHub

Useful commands:

    git status
    git add .
    git commit -m "chore: create initial Vite React TypeScript project"
    git push

Git is important because it allows the project to be reviewed, recovered and documented over time.

* * *

## GitHub

GitHub is the remote platform where the repository is stored.

Git works locally on the computer. GitHub stores a copy of the repository online.

This allows the project to be shared, reviewed and connected with deployment tools later.

The relationship is:

    Git = local version control
    GitHub = remote repository hosting

In this lesson, the project is connected to GitHub so future changes can be pushed as the course progresses.
